# UC12 Collaboration & Notifications

## Expression de besoin fonctionnelle
- **Objectifs** : améliorer la collaboration via notifications ciblées et intégrations chat.
- **Acteurs** : équipes delivery, managers, support.
- **Flux clés** : événements (tickets, builds) → règles de notification → canaux (Slack/Teams/Email) → actions.
- **Exceptions** : escalade en cas de SLA menacé; mode silence configurable.
- **Critères d’acceptation** : notifications configurables, intégrations chat, historisation.

## Analyse ROI
- **Gains** : réduction des temps de réaction, meilleure coordination, moins d’oubli.
- **Coûts** : intégrations canaux, gouvernance des règles.
- **KPI cibles** : temps de réponse moyen, taux de conversion notification→action, satisfaction utilisateurs.
